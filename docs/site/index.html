graphlib

matplotlib for modern C++. Same names, same object model, same rendering semantics — plt::plot(x, y, "r--o") does exactly what you think it does. Zero mandatory dependencies, byte-deterministic output, five platforms.

C++20 SVG · PNG · PDF · window mathtext date axes 10M points < 300 ms macOS · Linux · Windows (x64 + arm64) 
#include <graphlib/pyplot.hpp>
namespace plt = graphlib::pyplot;

int main() {
    std::vector<double> x = graphlib::linspace(0, 10, 200), y;
    for (double v : x) y.push_back(std::sin(v));
    plt::plot(x, y, "o-", {.label = "$\\sin(x)$"});
    plt::legend();
    plt::title("hello, $\\sqrt{graphlib}$");
    plt::savefig("hello.pdf");   // selectable text, drop into LaTeX
}

Why

  • Familiarity is the product. Names, defaults and behaviors mirror matplotlib at default settings; tick positions, autoscale margins, colormaps and date labels are pinned by fixtures generated from matplotlib itself.
  • Determinism is a feature. The same program produces byte-identical SVG and PDF on every platform — output files are golden-tested in CI.
  • The core stays dependency-free. C++20 STL plus vendored single-file libraries (AGG — matplotlib's own rasterizer — stb, DejaVu). GLFW windows are an opt-in.
